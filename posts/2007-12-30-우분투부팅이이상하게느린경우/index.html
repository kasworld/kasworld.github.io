우분투 부팅이 이상하게 느린경우

By SeukWon Kang

December 30, 2007

우분투 포럼을 뒤져서 알아낸 노우 하우를 기록 합니다.

이상하게 7.10이되면서 부팅하면서 나오던 이쁜 splash화면이 더이상 안나와서 그냥 그러러니 했었는데. 이렇게 화면이 나오지 않는 경우 스플래쉬 화면을 찾느라 부팅까지 느리게 만드는것 같습니다. ( 아마 계속 재시도 하느라 시간을 잡아 먹는듯. ) 그냥 스플래쉬 화면을 포기하고 예전처럼 부팅과정을 보는것을 감수 할생각이 있다면 ( 그리고 더이상 부팅시 필요없는 딜레이를 없애려면. ) sudo apt-get install sysv-rc-conf 하고 ( 부팅시 자동실행되는 서비스나 자동 실행 프로그램을 변경하는 프로그램입니다. ) sudo sysv-rc-conf 해서 usplash 의 자동 실행을 꺼주고 . ( 보통 부팅은 runlevel 2 이지만 잘모르겠으면 모두 꺼도 무방 ) sudo gedit /boot/grub/menu.lst 해서 title Ubuntu 7.10, kernel 2.6.22-14-generic root (hd0,5) kernel /boot/vmlinuz-2.6.22-14-generic root=UUID=e5044065-4424-40a1-a266-863da4b6b2b0 ro quiet splash initrd /boot/initrd.img-2.6.22-14-generic quiet 에서 splash 를 지운뒤 ( 디폴트 부팅 커널의 부팅 옵션을 바꿔주는 겁니다. ) ## additional options to use with the default boot option, but not with the ## alternatives ## e.g. defoptions=vga=791 resume=/dev/hda5 # defoptions=quiet splash 을 ## additional options to use with the default boot option, but not with the ## alternatives ## e.g. defoptions=vga=791 resume=/dev/hda5 ## defoptions=quiet splash 로 바꿔 줍니다. 그리고 재부팅해보면 됩니다.

( 각 커널 이미지 이름이나 부트 파티션은 제 시스템과 다를 수 있으니 바꾸시면 안됩니다. )
