리눅스에서 윈도우 프로그램이 필요한 경우

By SeukWon Kang

December 30, 2007

리눅스를 사용하더라도 어쩔수 없이 윈도우가 필요한 경우가 가끔 있을수 있습니다. 저같은 경우는 이에 대처하기 위해 여러가지 방법을 사용했는데요.

  1. 윈도우/리눅스간 멀티 부팅을 한다. 가장 일반적으로 많이 사용하는 방법이고 대부분의 경우에 처음 리눅스를 설치하면 택하는 옵션입니다. 저같은 경우는 새로 하드를 포멧하면서 파티션을 windows xp 32bit, winows xp 64bit , linux 64bit, linux swap , 데이타 저장및 교환용 파티션 이런 형태로 포멧을 했습니다. 데이터 교환용 파티션 같은 경우는 리눅스에선 윈도우 파티션을 읽고 쓸수 있기 때문에 없어도 대충 살만합니다만 정작 윈도우에선 리눅스 파티션을 읽고 쓸수 없기때문에 공용 데이타를 넣기 위한 공간으로 준비 했습니다. ( 지금은 윈도우로 부팅을 하지 않기 때문에 그냥 리눅스 파티션이 되어 버렸습니다. ^^ )

  2. 쓰다보면 재부팅 하는게 무지 귀찮기 때문에 슬슬 가상화 프로그램으로 눈이 돌아 갑니다. 리눅스에서 윈도우를 설치 부팅 할수 있는 가상기계는 3종 정도가 있는데요. 가장 유명한 vmware ( 상용입니다만 player 는 비상용이 되어 있습니다. ) 요즘 뜨고 있는 virtual box ( 리눅스쪽에선 가장 많이 추천 하고 있습니다. 공개 프로그램) 그리고 미래를 책임질것으로 기대를모으고 있는 Xen ( 전 아직도 이걸 어떻게 쓰는지 모르겠더군요.) 정도가 있습니다. 그런데 64bit 우분투에 디폴트로 설치 가능한 VM은 virtualbox 입니다. 이상하게 vmware player는 32bit 호스트 OS만 지원 하더군요. ( vmware 홈페이지 가서 64bit 플레이어를 받을 수있는지는 모르겠습니다만 비상용이긴 해도 공개 소프트가 아니라서 굳이 찾아볼 생각이 안들었습니다. ) 아무튼 이런 이유로 virtualbox를 설치했는데.. 기대와는 달리 기존에 존재하는 윈도우 파티션을 가상 머신 내부에서 부팅 하는 기능은 지원 하지 않더군요. 할수 없이 조그마한 가상 하드를 만들어 설치 했습니다. 설치하고 게스트 확장 드라이버를 설치하면 요즘 맥에서 유행하는 패러랠즈 같이 심리스 윈도우 기능도 지원해서 꽤 편하게 쓸만 합니다. ( 윈도우 프로그램이 VM 창안에 뜨는것이 아니라 리눅스 데스크 탑에 다른 일반 프로그램처럼 보이게 해주는 기능 )

  3. 제가 약간 특수한 경우겠지만 완전한 윈도우 실행 환경을 원한다면 저사양의 노트북을 하나 구하셔서 거기에 윈도우를 설치하고 리모트 데스크 탑으로 들어가서 사용하는것이 가능합니다. 이경우 화면과 키보드만 리눅스에서 보일뿐 나머지는 모두 윈도우 환경이기 때문에 완벽한(!) 호환성을 보입니다. ( –;; 퍽 당연하잖아. ) 이 당연한 이야기를 왜 하냐 하면 노트북인 경우 기계를 두대 운영하는것보다 소음/발열/전기/공간 면에 월등해서 부담이 없기 때문입니다. 설치된 노트북은 네트웍만 연결해서(무선이면 더 좋겠지요.) 적당히 눈에 안띄는곳에 박아두면 완벽 ^^ 파일도 공유폴더와 삼바로 연결하면 전혀 불편없이 사용할수 있습니다.

  4. 그리고 불안정하지만 가장 부담이 적은 wine을 사용하는게 가능합니다. 이 wine역시 우분투 기본 레포지토리에있는것은 64bit를 지원 하지 않는다고 반항하므로 wine 홈페이지에가서 설치 패키지를 받아서 사용하시면 됩니다. ( 정확히 말하면 wine 홈페이지에 레포지토리를 내 기계의 패키지 검색 레포지토리로 등록하면 됩니다. )

제가 우분투를 설치한후 위의 4단계를 순서대로 거쳐 왔습니다. ^^ ( 오픈 소스가 아닌 꼭써야하는 윈도우 프로그램이 필요해서 말이죠. ;; )
