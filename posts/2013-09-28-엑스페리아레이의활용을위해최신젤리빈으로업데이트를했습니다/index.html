엑스페리아 레이의 활용을 위해 최신 젤리빈으로 업데이트를 했습니다.

By SeukWon Kang

September 28, 2013

오래전에 펌웨어를 ics로 업데이트 하고 옵티이이로 바꾼후에는 책상속에서 얌전히 잠자고 있던 레이를 꺼낼 생각이 든 것은 크기 때문입니다.

간단히 산책을 나가거나 운동을 할 때 주머니속 옵티이이는 생각보다 꽤 부담이 됩니다.

게다가 요즘은 gboard 8.9를 주로 사용하며 핸드폰은 주로 테더링 용으로 사용하니 더욱더 레이 생각이 나더군요.

그래서 예전 기억을 더듬어서 레이 부활을 시도 했습니다.

일단 예전에 제가 써둔 블로그 글을 읽어보고 (기억을 되살리는 겸.. ) http://kasw.blogspot.kr/search/label/%EB%A0%88%EC%9D%B4

내용이 별게 없군. 하고 생각하면서.. ;;;

어쩔수 없지 뭐 하고 

http://forum.xda-developers.com/showthread.php?t=1628264 를 들어가서 사용할 커펌을 골랐습니다.

좀 안보던 사이(1년 2개월 정도)에 커펌 종류가 엄청나게 늘었더군요.

일단 제일 유명한 수퍼젤리빈을 선택했습니다. http://forum.xda-developers.com/showthread.php?t=2135918

설치 순서가 나와 있긴 하지만 무시하고 ;; 메모리 카드에 롬을 복사해 넣고 리커버리 모드로 부팅해서 superjellybean 6.0을 설치했습니다.

당연하다는 듯이 부팅에 실패하는군요. ;;;;

깊이 반성하면서.. ( 역시 순서를 지켜야해.. )

그럼 (하라는 대로 ) 커널을 올려야 겠네.. 라고 생각하고 ( 생각해보면 ics 올릴때도 똑같은 삽질을 했던 기억이 나는 군요. ;; )

추천 커널인 fusion kernel 5.4 을 올릴 준비를 합니다.

fastboot를 사용해서 설치하라고 하는데 그게 뭐였는지 전혀~~ 기억이 나지 않습니다. ;;

전에 설치할때 썼던 디렉토리 파일 들을 보니 flashtool 어떠구하는 설치 파일들이 보이기에 저걸로 설치했었나? 하고 긴가 민가 하고 일단 윈도우로 부팅했습니다. ( 전 집에서도 거의 linux만 사용합니다. )

flashtool을 설치하고 실행하니 디바이스(레이 기계)를 못찾습니다.

이전에도 드라이버 설치에 고생했다고 전에 쓴글에 글에 있는데 이번엔 아무리 해도 기계를 찾지를 못하더군요.

한 2시간 정도 헤메다가 열받아서 그냥 리눅스로 부팅했습니다.

그리고 밑져야 본전이지 하고 synaptic에서 fastboot를 검색해보니 패키지가 있기에 설치했습니다.

윈도우용 flashtool을 GUI 툴인데 fastboot를 명령행용 프로그램입니다.

실행해보니 레이를 잘 찾는 군요..  그리고 커널을 어떻게 올리나 검색해서 5분만에 커널 설치를 완료 했습니다.

역시 윈도우에서는 안 되는게 리눅스에선 간단히 되는군요. ^^;;

참고로 ( 미래의 나를 위해서 ;; ) 커널 설치 명령어는 기계가 보이는지 확인 fastboot devices 커널을 설치 ( 커널 파일 이름이 urushi_fusion_5.4.img 입니다 ) fastboot flash boot urushi_fusion_5.4.img 기계 재부팅 fastboot reboot 입니다.

fastboot모드로 진입하는 방법은 기계가 꺼진 상태에서 볼륨 위쪽 버튼을 누른 채로 pc 와 usb 케이블로 연결하면 됩니다. 홈 버튼 주변 이 파란색으로 빛나면 재대로 진입한 것입니다.

이후에 추가 app ( gapps-jb-20130624-sjb 4.1.2.zip )을 리커버리 모드에서 설치하면 끝입니다.

그런데 정작 옵티이이에서 사용하던 심카드가 인식이 안되는군요. ;;; 분명히 원래 레이에서 쓰던걸 마이크로 심으로 잘라서 옵티이이에서 사용하고 있는 것일텐데 말이죠..
