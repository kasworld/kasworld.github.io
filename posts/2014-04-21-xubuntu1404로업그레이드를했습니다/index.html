xubuntu 14.04 로 업그레이드를 했습니다.

By SeukWon Kang

April 21, 2014

신경 쓰지 못하고 있었는데 작업중 컴퓨터에서 며칠 전 14.04로 업그레이드 하라는 메시지가 나오길래 아. 어느새 릴리즈 되었군 하고 깨달았습니다. 

작업 머신의 업그레이드는 일단 뒤로 미루고 ( 하던 작업을 계속 해야 하니 ) 일단 옆에 있는 노트북부터 업그레이드를 실행했습니다. 

( 데이터는 모두 백업 되어 있어 문제가 생기면 포맷하고 새로 설치하면 되니 연습용으론 최적입니다. )

배포판 업그레이드는 많아야 6개월에 한번 진행하는 것이니 명령어를 항상 까먹습니다. 

sudo do-release-upgrade 

를 사용합니다. 

한참 목록을 다운 받고 나오는 메시지 대로 진행하면 1~2시간 내에 모든 업그레이드를 완료해줍니다. 

시스템 설정중 본인이 변경한 설정들이 있으면 중간 중간 보존할지 말지를 물어보니 다른 일을 하다 보면 질문의 답을 기다리는 커서가 깜빡이는 화면을 볼 수 있습니다. 

( 그러면 더 오래 걸리게 되겠지요. ^^ )

노트북을 아무 문제 없이 업그레이드 하고 나서 “역시 문제 없군”, “그리고 달라진 것도 거의 없어 보이네.” 하고 생각하면서 주 작업 머신의 업그레이드를 진행했습니다. 

( 내부적으론 많이 바뀌었겠지만 사용하고 있는 설정을 거의 완벽하게 유지 해주니 바뀐점을 느끼기 힘듭니다. ^^)

주 작업 머신의 업그레이드 역시 전혀 문제없이 완료되기에(심지어 전 3 모니터를 사용하고 있습니다. ) 뭔가 바뀌었나 하고 이런 저런 곳을 찾아보고 벤치마크 프로그램을 실행해보고 있는데 3D쪽이 좀 느려진 느낌이 듭니다. 

어라? 하고 그래픽 드라이버를 살펴보니 제가 수동으로 설치해놓은 NVIDIA-Linux-x86_64-337.12가 아니고 xubuntu 기본인 nvidia-331 드라이버가 설치되어 있습니다. 

아마도 337.12는 정식 릴리즈가아닌 beta드라이버니 최신 안정 버전을 설치한 것일 겁니다. 

예전 기억을 되살려 드라이버를 다시 설치했습니다. 

그리고 아무생각없이 재부팅.. 

…

부팅이 안되는군요. .잠시 경직…

곰곰히 생각해보니 예전 드라이버를 그냥 둔 채로 새 드라이버 설치를 진행 했나봅니다. 

일단 부팅은 시켜야 하니 

recovery모드로 부팅해서 ( 정확히는 이전 버전 커널로 부팅했습니다. )

http://www.yourownlinux.com/2014/04/how-to-install-nvidia-337-12-graphics-driver-in-linux.html

에 나온대로 

sudo apt-get purge nvidia*

를 하고 재부팅.. 

무사히 화면이 나오는 것을 보고 가슴을 쓸어 내렸습니다. 

요즘엔 리눅스를 사용하면서 문제가 생기는 경우가 워낙 적어서 아주 오랫만에 당한 부팅 불가 였습니다.

그리고는 14.04를 설치하면 적용하려고 했던 intel_pstate

http://www.webupd8.org/2014/04/prevent-your-laptop-from-overheating.html

을 적용했습니다. 

일단 제 샌디브릿지 주 작업 머신에는 잘 적용된다고 해서 문제 없이 적용했지만 

하스웰에선 될지 안될지 모른다고 해서 하스웰이 들어 있는 노트북에는 아직 적용하지 않은 상태 입니다. 

그리고 전 xubuntu를 사용하기때문에 별 상관 없어서 적용하지 않았지만 ubuntu를 사용한다면
